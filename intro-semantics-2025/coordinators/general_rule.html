A general rule

\[ \newcommand{\expr}[3]{\begin{array}{c} #1 \\ \bbox[lightblue,5px]{#2} \end{array} ⊢ #3} \newcommand{\ct}[1]{\bbox[font-size: 0.8em]{\mathsf{#1}}} \newcommand{\abbr}[1]{\bbox[transform: scale(0.95)]{\mathtt{#1}}} \def\true{\ct{T}} \def\false{\ct{F}} \]

To explain these facts, it would be useful to have a systematic account of the meanings of and and or. Let’s start by providing the sentence-coordinator meanings in terms of some lexical entries. For each lexical entry, we’ll annotate the word itself with a category name, just to keep coordinators of different categories distinct (e.g., we’ll call the sentence-level and \(\textit{and}_{s}\) and the sentence-level or \(\textit{or}_{s}\)).

Sentence coordinators

Let’s start with sentences.

    1. \(⟨\textit{and}_{s}, (λa.(λb.a = b = \true))⟩ ⊢ ((s\backslash s)/s)\)
    2. \(⟨\textit{or}_{s}, (λa.(λb.¬(a = b = \false)))⟩ ⊢ ((s\backslash s)/s)\)

Verb phrase coordinators

What about the versions of and and or when they coordinate verb phrases? Well, they should both have the same syntatic category:

\[(((np\backslash s)\backslash(np\backslash s))/(np\backslash s))\]

Or in tree notation:

        (/)
        / \
       /   \
      /     \
    (\)     (\)
    / \     / \
   /   \   np  s
 (\)   (\)
 / \   / \
np  s np  s

As a result, they should also both have the following semantic type:

\[((e → t) → ((e → t) → (e → t)))\]

Or in tree notation:

       ─>
      / \
     /   \
    /     \
   ─>      ─>
  / \     / \
 e   t   /   \
        /     \
       ─>      ─>
      / \     / \
     e   t   e   t

Thus their meanings should each take two arguments in order, where both of these arguments are characteristic functions of sets of entities (functions of type \((e → t)\)); and they should each give back a new characteristic function. The lexical entries in (2) accomplish this.

    1. \(⟨\textit{and}_{(np\backslash s)}, (λf.(λg.(λx.f(x) = g(x) = \true)))⟩ ⊢ (((np\backslash s)\backslash(np\backslash s))/(np\backslash s))\)
    2. \(⟨\textit{or}_{(np\backslash s)}, (λf.(λg.(λx.¬(f(x) = g(x) = \false))))⟩ ⊢ (((np\backslash s)\backslash(np\backslash s))/(np\backslash s))\)

Transitive verb coordinators

What about when and and or coordinate transitive verbs? Transitive verbs have the syntactic category \(((np\backslash s)/np)\), so and and or should have the following syntactic category in this case:

\[((((np\backslash s)/np)\backslash((np\backslash s)/np))/((np\backslash s)/np))\]

Or in tree notation:

               (/)
               / \
              /   \
             /     \
            /       \
           /         \
          /           \
        (\)           (/)
        / \           / \
       /   \        (\)  np
      /     \       / \
    (/)     (/)    np  s
    / \     / \
  (\)  np (\)  np
  / \     / \
 np  s   np  s

Using the usual mapping from categories to types, we can see that they should therefore have the following semantic type:

\[((e → (e → t)) → ((e → (e → t)) → (e → (e → t))))\]

Or in tree notation:

        ─>
       / \
      /   \
     /     \
    /       \
   /         \
  ─>          ─>
 / \         / \
e  ─>       /   \
  / \      /     \
 e   t    ─>      ─>
         / \     / \
        e   ─>  e   ─>
           / \     / \
          e   t   e   t

Again, their meanings each take two arguments in order, but now both of the arguments are functions of type \((e → (e → t))\)—i.e., functions thta take two entities (in order) and give back a truth value. The lexical entries in (3) assign them meanings that do this.

    1. \(⟨\textit{and}_{((np\backslash s)/np)}, (λf.(λg.(λx.(λy.f(x)(y) = g(x)(y) = \true)))⟩ ⊢ ((((np\backslash s)/np)\backslash((np\backslash s)/np))/((np\backslash s)/np))\)
    2. \(⟨\textit{or}_{((np\backslash s)/np)}, (λf.(λg.(λx.(λy.¬(f(x) = g(x) = \false)))))⟩ ⊢ ((((np\backslash s)/np)\backslash((np\backslash s)/np))/((np\backslash s)/np))\)
